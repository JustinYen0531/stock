# UI Review Checklist

Use this checklist when a UI answer needs a quick final pass.

## Flow

- Confirm the main user action is obvious within the first viewport.
- Confirm the layout supports scanning before detailed reading.
- Confirm copy uses realistic labels, headings, and button text.

## Visual System

- Confirm the page has one clear visual direction instead of mixed styles.
- Confirm primary, neutral, success, warning, and error colors are defined when relevant.
- Confirm contrast is strong enough for WCAG 2.1 AA use cases.

## Responsive Behavior

- Confirm the most important content appears early on mobile.
- Confirm grid sections stack cleanly without overcrowding.
- Confirm tap targets and spacing remain comfortable on small screens.

## Components

- Confirm buttons, inputs, cards, and navigation have clear states when they matter.
- Confirm component descriptions are concrete enough for implementation.
- Confirm recommendations fit the existing design system when one exists.

## Implementation

- Confirm the Tailwind snippet reflects the described layout and style.
- Confirm dark mode is addressed when appropriate.
- Confirm the snippet focuses on the important screen rather than unnecessary boilerplate.
