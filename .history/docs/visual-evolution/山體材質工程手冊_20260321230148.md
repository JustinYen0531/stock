# 山體材質工程手冊 (Terrain Material Engineering)

本手冊定義如何將視覺演化框架的設計，具體落實為 Canvas 渲染代碼，確保山體具備豐富的細節與動態滾動效果。

---

## 1. 渲染層級 (Layering Architecture)
每一幀的渲染必須遵循以下 Z-Index 順序：

1.  **Layer 0 (-100)**：遠景 Vista（視差比例 0.1 ~ 0.2）。
2.  **Layer 1 (-50)**：山體底層裝飾（填充 `TEXTURE` 的遠山背景）。
3.  **Layer 2 (0)**：主地形填充。
    *   **Solid Base**：產業代表色底層。
    *   **Texture Tile**：動態滾動的無縫紋理（Overlay 模式）。
    *   **Sheen Layer**：山頂背光光輝。
4.  **Layer 3 (+50)**：主地形邊緣（Edge Treatment / Rim Light）。
5.  **Layer 4 (+100)**：特製地圖組件（Landmark Props）與粒子系統。

---

## 2. 山體填充演算法 (The Fill Logic)
山體必須透過 `clip()` 進行填充，而非直接畫線。

### 步驟 A：建立路徑 (Path)
*   計算 K 線所有座標點，並形成一個連到底部側邊的閉合路徑。

### 步驟 B：材質疊加 (Multi-Layering)
```javascript
// 邏輯示意
ctx.clip(fillPath);
ctx.fillStyle = theme.baseColor; // 1. 實色底
ctx.fillRect(W, H);

ctx.globalCompositeOperation = 'overlay'; // 2. 紋理疊加
ctx.fillStyle = ctx.createPattern(textureImg, 'repeat');
ctx.translate(-terrainScrollX, 0);
ctx.fillRect(W, H);

ctx.globalCompositeOperation = 'screen'; // 3. 光澤層
ctx.fillStyle = sheenGradient;
ctx.fillRect(W, H);
```

---

## 3. 技術參數標準 (Implementation Standards)

| 參數 | 標準值 | 用法 |
| :--- | :--- | :--- |
| **TEXTURE_SIZE** | 256x256 / 512x512 | 紋理切片必須是 2 的方冪，方便 CanvasPattern 重複。 |
| **PATTERN_ALPHA** | 0.45 ~ 0.75 | 確保在深色底山體上紋理依然可見。 |
| **SHEEN_HEIGHT** | 40px ~ 80px | 從山脊線向下延伸的光輝範圍。 |
| **EDGE_WIDTH** | 3px ~ 12px | 山脊發光線的寬度，隨漲跌情緒改變。 |
| **PARALLAX_RATIO** | 0.14 | 紋理滾動與相機移動的預設視差比例。 |

---

## 4. 數據驅動參數清單 (Configuration)
渲染器應接受以下 JSON 結構：
*   `fill_texture`: 紋理圖片 Key。
*   `texture_density`: 紋理重疊密度。
*   `edge_style`: ['ice', 'soft', 'hard', 'tape', 'glitch']。
*   `glow_color`: 山脊發光 HSL。
*   `sprite_pack`: 對應的產業 Landmark 組件包。

---

## 5. 常見問題修復
*   **山體灰灰的**：檢查 `sheen` 層是否呼叫了 `fillRect`，以及 `globalAlpha` 是否過低。
*   **動畫卡頓**：確保 `createPattern` 在切換主題時才新建，而非每一幀新建。
*   **紋理閃爍**：`terrainScrollX` 代入時需取整數或對 `TEXTURE_SIZE` 取餘數。
